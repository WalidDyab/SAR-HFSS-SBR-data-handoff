Executive summary. This package hands off completed, physically grounded HFSS SBR+ complex monostatic RCS results for a sphere, cone, dihedral reflector, and trihedral reflector. The complex values are intended for downstream phase-history preparation and SAR/inverse-SAR processing; they are not final SAR images.

1. Simulation philosophy

HFSS SBR+ is used here to efficiently model high-frequency scattering and ray interactions, including the multiple-bounce mechanisms that matter for corner reflectors. HFSS supplies the electromagnetic response over incident angle and frequency; the downstream team supplies aperture selection, calibration, gridding, windowing, and image formation.

Keep the phase. Use the exported real and imaginary values. Plotted dB curves are diagnostics; magnitude-only traces do not preserve the phase needed for coherent SAR/inverse-SAR formation.

2. How to use this package

1
Begin with the master slice.

Use four_cases_dep30_sar_ready_master_table.csv for an initial common-depression demonstration.

2
Build the complex aperture.

For Phi, form S = s_phi_real + j*s_phi_imag and reshape by target, aspect, and frequency.

3
Process coherently.

Apply the agreed convention, calibration/reference treatment, aperture selection, windowing, and image-formation method.

4
Return to raw data when needed.

Use each full CSV for another depression slice, the full angular response, or Phi/Theta vector processing.

Convention used in this report and derived file
aspect_deg = (IWavePhi - 180) mod 360
depression_deg = IWaveTheta - 90

Thus IWaveTheta = 120° maps to depression_deg = 30°. Confirm this convention against downstream scene/platform geometry before interpreting image orientation.

6. Diagnostic plots

The existing figures remain linked. They support response comparison and slice selection; the complex CSV values remain the processing input. Treat dB magnitude as the stated transform of the complex component magnitude (for example 20 log10(|complex response|)), not as an independently exported absolute dBsm field.

8. Opening and sharing

  1. Keep index.html, data/, derived/, assets/, and images/ together.
  2. Open index.html in a modern browser. A small local server is optional for pre-distribution review.
  3. Zip the entire package folder, not the HTML alone.
  4. After extraction, preserve the folder structure so images and downloads continue to work.
